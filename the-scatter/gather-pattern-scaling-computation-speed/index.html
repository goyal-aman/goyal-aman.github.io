The Scatter/Gather Pattern: Scaling Computation Speed

31 Jan, 2026

Core Concept

  • Scatter/Gather = parallel processing pattern
  • A request is scattered to multiple nodes and the partial results are gathered and combined into one response
  • Goal: Reduce latency (faster response times)

When to Use

  • Best for “embarrassingly parallel” problems that are independent and parallelizable, such as search queries, data aggregation, and analytics

Key Challenges

  • Diminishing returns: Network and coordination costs eventually outweigh the benefits of adding more nodes
  • Straggler problem: The system speed is dictated by the slowest node; more nodes increase the chance of a slowdown
  • Reliability: If one node fails, the entire request may fail

Solutions

  • Replicate each shard: Use multiple replicas for each shard to provide fault tolerance and load balancing
  • Smart scaling: Balance parallelism against coordination overhead

Trade-offs

  • Pros: Faster response times and efficient handling of compute-heavy tasks
  • Cons: Complex coordination, network overhead, and sensitivity to “tail latency” (slowest components)

One-line Summary

Scatter/Gather distributes a request across multiple nodes for parallel processing and combines results to reduce latency.
