AI Inference and Serving

21 Mar, 2026

Core Concept

  • AI systems revolve around:
    • Model (trained weights)
    • Inference (runtime prediction)
  • Inference is typically exposed as a REST API

Key Building Blocks

1. Model vs Training vs Inference

  • Training → expensive, offline
  • Fine-tuning → cheaper
  • Inference → real-time usage

2. Prompt Engineering

  • Input wrapped in prompt
  • Adds context, instructions, personalization

Hosting Models

  • Latency and compute are key constraints

Deployment options

  • Cloud inference
  • Self-hosted

Small Language Models (SLMs)

  • Smaller, faster, cheaper
  • Lower accuracy trade-off

Model Distribution

  • ONNX format recommended
  • Use model hubs
  • Cache models locally

Safe deployment

  • Gradual rollout (canary)

Development Strategy

  • Use smaller models in dev/test
  • Larger models in prod

Retrieval-Augmented Generation (RAG)

Flow:

User Query → Fetch data → Augment prompt → Model

Testing AI Systems

  • Not exact match
  • Measure statistical quality

Deployment & Evaluation

  • Track user satisfaction
  • Track output quality

One-line Summary

AI inference systems serve trained models via APIs, where prompt engineering, efficient model hosting, and techniques like RAG are key to delivering accurate, scalable, and cost-effective intelligent applications.
